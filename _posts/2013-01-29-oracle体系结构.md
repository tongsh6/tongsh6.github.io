Oracle 数据库由物理文件和实例（`instance`）组成，用户只能通过实例与数据库交互。

## 整体示意

```text
                    INSTANCE
--------------------------------------------------
| --------------         SGA   <--- 系统全局区    |
| |-shared pool-|                    redo log     |
| | library     ||    database buffer cache     | R
| | cache       ||                    streams    | A
| | dictionary  ||    java pool / large pool    | M
| |_|_cache_____||______________________________ |
| PMON SMON DBWR LGWR CKPT OTHERS FBAR          |
--------------------------------------------------

DISK：控制文件 / 数据文件 / 重做日志 / 参数文件 / 口令文件 / 归档文件 / 警告文件
```

## 几个基础概念

- `block`：数据库读写数据的最小单元，常见为 `8k`。
- 脏数据：缓存中的数据和数据文件中的数据不一致，已经 `commit`，被 `redo log buffer` 接受，但尚未写入 `data file`。
- `checkpoint`：定期执行的任务，促使缓存中的数据刷回物理文件。
- `SCN`（system change number）：记录数据库系统的变更顺序，会写入物理文件头。

查看块大小：

```sql
show parameter db_block_size;
```

## 1. Instance

### 1.1 SGA（System Global Area）

#### 共享池（shared pool）

- 库缓存（Library Cache，LRU 列表）
  - 保存最近执行过的 SQL 文本、解析后的代码和执行计划。
  - 如果命中缓存，可直接执行。

  ```sql
  select * from v$sqltext where sql_text like '%select * from emp%';
  ```

- 数据字典缓存区（Data Dictionary Cache）
  - 保存数据库中的数据字典信息，例如表名、列名、登录用户、权限、方案等。
  - 内容和大小会随着数据库运行动态变化。
  - 常见视图有：`user_objects`、`user_tables`、`dba_tables` 等。

#### 数据高速缓存区（Database Buffer Cache）

- 存放最近最常使用到的数据。
- DML 操作的数据会在 DBWR 进程空闲时写入 `data files`。
- 包括脏数据缓存区（dirty buffer）和空闲缓存区（free buffer）。
- 正在被操作的缓冲区（pinned buffer）既可能是脏区，也可能是空区。

#### 重做日志缓存区（Redo Log Buffer）

- 记录 DDL / DML 操作，作为变更日志。
- Redo Log Buffer 中的数据最终会写入 redo log files。

#### 其他内存区

- `java pool`：Java 程序所需的内存空间。
- `large pool`：可选，查询、排序、备份、恢复可能会用到。
- `stream pool`：10g 以后用于大批量移动、复制。

### 1.2 后台进程

- `DBWR`（Database Writer）
  - 把数据缓存写到数据文件。
  - 进程数最多可配置到 20 个，参数：`db_writer_processes`。
  - 常见触发时机：
    1. 系统 checkpoint 事件
    2. 缓存区脏块比例达到预设值
    3. 三秒超时
    4. `drop`、`truncate`
    5. 表空间变动等其他操作

- `LGWR`（Log Writer）
  - 把日志缓存写到日志文件。
  - 常见触发时机：
    1. `commit`
    2. redo log buffer 达到 `1/3`
    3. redo log buffer 超过 `1M`
    4. 三秒超时

- `CKPT`（Checkpoint Process）
  - 把上一个检查点之后的数据同步到物理文件，并更新文件头。

- `PMON`（Process Monitor）
  - 清理失败进程，释放资源。

- `SMON`（System Monitor）
  - 负责实例恢复，清理 SGA 中不再使用的临时空间。

- `ARCH`（Archive Process）
  - 数据库处于归档模式时，把日志文件备份为归档文件。

  ```sql
  archive log list;
  ```

- `FBAR`（Flashback Archiver Writer）
- `OTHERS`

### 1.3 PGA（Program Global Area）

每个用户进程对应一个私有区域，保存变量、代码等运行时数据。

## 2. 物理文件

- 控制文件（control files）
  - 二进制文件。
  - 保存数据库名称、创建时间、数据文件位置和名称、当前日志序列号、物理结构等。
  - 最好在不同介质上放置两个以上副本。

- 重做日志文件（redo log files）
  - 记录数据库发生过的变化。
  - 是实例恢复的重要保障。
  - 在归档模式下，归档完成后才会被覆盖。
  - 最好在不同介质上放置两个以上副本。

- 数据文件（data files）

- 归档文件（archive file）
  - 重做日志文件的备份。
  - 在归档模式下，归档完成后，重做日志文件才会被覆盖。

- 参数文件（parameter file）
  - 不可改：`db_name`、`sid`、`db_block_size`
  - 可改：`open_cursor` 等
  - `pfile.ora`：静态参数文件，文本
  - `spfile.ora`：服务器参数文件，二进制

- 口令文件（password file）
  - 用于验证特权用户权限的二进制文件。
  - 支持口令验证、操作系统验证、文件验证。

- 警告文件（alert file）
  - 记录数据库启动关闭、数据文件增删、表空间变更等信息。

## 3. 物理结构和逻辑结构

- 物理结构
  - 数据文件
  - 操作系统块

- 逻辑结构
  - 表空间（tablespace）
  - 段（segments）
  - 区（extends）
  - 块（blocks）

- 一个表空间包含多个数据文件。
- 一个块包含多个操作系统块。

## 4. 数据字典

数据字典用于记录数据库本身的信息。

### 4.1 命名规则

- `user_***`：用户自己方案下的对象
- `all_***`：当前用户可访问的对象
- `dba_***`：数据库中的全部对象（需要 DBA 权限）
- `v$***`：动态性能视图

### 4.2 常用数据字典

```sql
select table_name from dict where table_name like '%CONSTRAINT%';
```
