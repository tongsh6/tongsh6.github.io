一道典型的线程协作面试题：循环按顺序打印 `A B C D`，每个线程只负责打印一个字母。

## 思路

这里的实现思路是：

- 用一个共享数组保存打印顺序
- 用 `AtomicInteger` 记录当前应该打印到哪个下标
- 每个线程只负责自己对应的字母
- 通过 `synchronized + wait/notifyAll` 协调线程顺序

## 示例代码

```java
import java.util.concurrent.atomic.AtomicInteger;

/**
 * 一道面试题，循环按顺序打印ABCD，每个线程只打印一个字母
 */
public class Print {
    public static void main(String[] args) {
        String[] contents = { "A", "B", "C", "D" };
        AtomicInteger index = new AtomicInteger(0);

        for (String content : contents) {
            new Thread(new Printer(index, contents, content)).start();
        }
    }
}

class Printer implements Runnable {
    private final String[] content;
    private final AtomicInteger index;
    private final String printText;

    public Printer(AtomicInteger index, String[] content, String printText) {
        this.content = content;
        this.index = index;
        this.printText = printText;
    }

    @Override
    public void run() {
        while (true) {
            synchronized (content) {
                while (!printText.equals(content[index.intValue()])) {
                    try {
                        content.wait();
                    } catch (InterruptedException e) {
                        throw new RuntimeException(e);
                    }
                }
                System.out.println(printText);
                index.set(index.intValue() + 1);
                if (index.intValue() >= content.length) {
                    index.set(0);
                }
                content.notifyAll();
            }
        }
    }
}
```

## 关键点

- `while (!printText.equals(...))` 不能写成 `if`，否则线程被唤醒后可能出现误判
- 所有线程都在同一个锁对象 `content` 上等待和唤醒
- 每次打印完成后把索引往后推进，超出长度后重置为 `0`
- 使用 `notifyAll()` 而不是 `notify()`，避免只唤醒到错误线程

## 可以继续优化的地方

- 当前实现会无限打印，没有退出条件
- `AtomicInteger` 在这里不是必须的，因为已经放在 `synchronized` 代码块里
- 更现代一点的写法可以考虑 `ReentrantLock + Condition`

## 小结

这类题的重点不在于 API 用得多花哨，而在于是否真正理解：

- 谁有资格打印
- 不该打印的线程如何等待
- 打印完成后如何唤醒下一位
- 为什么要防止虚假唤醒
